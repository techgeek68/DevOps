### Lab FM2.A: Identify Network Basics on Your Own Machine

**Objective**

Get comfortable looking at real network information on your own computer before doing any of this inside a VM. This lab uses only your everyday laptop or desktop, no VM required.

**What You Need**

- Your own laptop or desktop computer
- No installation required

**Prerequisites**

Read Sections FM2.3 and FM2.4 before starting.

**Instructions**

Find your own IP address. On Windows, open Command Prompt and run `ipconfig`. On macOS or Linux, open a terminal and run `ifconfig` or `ip addr show`. Locate the IP address assigned to your main network connection. It will be a private address, typically starting with `192.168.`, `10.`, or `172.16.` through `172.31.`.

Determine whether this address is public or private using what you learned in Section FM2.3.

Visit a website that shows you your public IP address, such as `whatismyipaddress.com`, using your browser. Write down the address it shows you.

Compare the two addresses you found. They should be different. Write two or three sentences explaining why they are different, using the public versus private distinction from Section FM2.3.

Open a terminal or command prompt and run a basic lookup for a website name. On any system, this command works: `nslookup github.com`. Record the IP address it returns.

---
**Expected Output**

A short written note covering: your private IP address, your public IP address, a brief explanation of why they differ, and the IP address that `github.com` resolved to.

---
**Common Issues**

If `ipconfig` or `ifconfig` shows multiple network adapters, look for the one that is actually connected, usually labeled Wi-Fi, Ethernet, or similar, and ignore virtual adapters with names like VirtualBox or VPN unless you know you are using one.

### Lab FM2.B: Hypervisor Type Identification

**Objective**

Build the habit of correctly classifying virtualization software as bare metal or hosted, since this distinction comes up constantly once you start working with real infrastructure.

**What You Need**

- Internet access for research
- A text editor or document

**Prerequisites**

Read Section FM2.2 before starting.

**Instructions**

For each of the following, research and determine whether it is a bare metal hypervisor or a hosted hypervisor. Write one sentence justifying each answer based on what you learned in Section FM2.2.

- VMware ESXi
- VirtualBox
- Microsoft Hyper-V
- VMware Workstation Pro
- KVM
- Parallels Desktop

After classifying all six, answer this question in two or three sentences: if a company is building the actual infrastructure for a production application that thousands of people will use, which type of hypervisor would they most likely choose, and why does the answer connect back to what you learned about performance and overhead.

**Expected Output**

A table or list with all six items correctly classified, one justification sentence per item, and your written answer to the final question.

### Lab FM2.C: Trace a Request in Plain English

**Objective**

Practice describing, step by step and in your own words, everything that happens between typing a website address and seeing the page appear. This builds the mental model you will rely on for every networking lab in Module 2.

**What You Need**

- Paper or a text editor
- No terminal or special tools required

**Prerequisites**

Read all of Sections FM2.3, FM2.4, FM2.5, and FM2.6 before starting.

**Instructions**

Imagine you type `www.example.com` into your browser and press Enter. Write a clear, numbered, step by step explanation of everything that happens from that moment until the webpage appears on your screen.

Your explanation must include all of the following concepts somewhere in the sequence, in the correct order: DNS resolution turning the name into an IP address, the request traveling across the network to reach the destination, a firewall on the destination server deciding whether to allow the request through, and, if the website uses more than one server behind the scenes, a load balancer deciding which server should handle this particular request.

Write this as though you were explaining it to a friend who has never studied networking. Avoid copying definitions directly from this module. Use your own words.

---
**Expected Output**

A numbered, step by step explanation, written in your own words, correctly incorporating DNS, network firewall behavior, and load balancing in the right order.

---
**Common Mistakes to Avoid**

Do not skip the firewall step. Many learners jump straight from DNS resolution to the page loading and forget that the destination server's firewall is part of the decision chain. Do not assume every website uses a load balancer. Mention it as something that happens specifically when more than one server is involved, not as a universal step in every single request.
