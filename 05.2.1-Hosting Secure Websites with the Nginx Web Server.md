# Nginx HTTPS Configuration: SSL/TLS Security & Performance Optimization

- HTTPS encrypts communication between web browsers and servers using SSL/TLS certificates, protecting data from interception and verifying server identity.
  
- SSL/TLS encrypts data in transit between client and server.
  
- TLS 1.2+ is required for modern, secure deployments (TLS 1.0 and 1.1 are deprecated and disabled by most browsers and servers).

- Session caching reuses negotiated SSL session parameters, avoiding a full handshake on subsequent connections and reducing CPU overhead significantly (commonly cited at 80/90%, though actual savings vary by traffic pattern and hardware).
  
- HTTP/2 multiplexes multiple requests over a single connection, reducing latency and improving throughput.
  
- Online Certificate Status Protocol (OCSP) stapling allows Nginx to fetch and cache the certificate's revocation status from the CA, attaching it to the TLS handshake so clients don't need to perform a separate OCSP lookup.
  
- Proper cipher selection balances security strength against performance and compatibility.

**SSL/TLS:**

- Cryptographic protocols for secure communication. SSL (versions 2.0 and 3.0) is fully deprecated and insecure; TLS (currently 1.2 and 1.3) is the modern standard.

**Certificate:**

- A digital document that binds a public key to a server's identity, enabling clients to verify they're talking to the right server.

**Certificate Authority (CA):**

- A trusted organization that issues and signs certificates (e.g., Let's Encrypt, DigiCert, Sectigo).

**SSL/TLS Certificate:**

- **Let's Encrypt:**
  - Free, automated certificates (90-day validity, designed for automated renewal via Certbot or ACME clients)
    
- **Commercial CAs:**
  - Paid certificates with extended validation (EV) and organization validation (OV) options; some offer longer validity periods
    
- **Self signed:**
  - For testing/development only, browsers will display security warnings since no trusted CA has signed them

**Basic Nginx SSL Configuration**

```nginx
server {                                                                                    # Begin server block for HTTPS
    # Enable SSL and HTTP/2
    listen 443 ssl http2;                                                                   # Listen on port 443 with SSL/TLS and HTTP/2 protocol enabled
    server_name example.com;                                                                # Domain name this server block responds to

    # Configure certificates
    ssl_certificate /etc/nginx/ssl/cert.pem;                                                # Path to SSL certificate file (public key + full certificate chain)
    ssl_certificate_key /etc/nginx/ssl/key.pem;                                             # Path to SSL private key file (keep this secure and restrict permissions)

    # Set protocols and ciphers
    ssl_protocols TLSv1.2 TLSv1.3;                                                          # Only allow TLS 1.2 and 1.3 (disables older, insecure versions)
    ssl_ciphers HIGH:!aNULL:!MD5:!3DES;                                                     # Use strong cipher suites; exclude null, weak, and legacy ciphers
    ssl_prefer_server_ciphers on;                                                            # Server selects cipher order for TLS 1.2; has no effect on TLS 1.3 (which manages its own cipher selection)

    # Enable session cache
    ssl_session_cache shared:SSL:10m;                                                        # Shared memory cache for SSL sessions (10 MB stores approximately 40,000 sessions at ~256 bytes each)
    ssl_session_timeout 10m;                                                                 # SSL session remains valid for 10 minutes before requiring a new handshake
    ssl_session_tickets off;                                                                 # Disable session tickets to preserve Perfect Forward Secrecy (PFS); compromised ticket keys can otherwise decrypt past sessions

    # Configure OCSP stapling
    ssl_stapling on;                                                                          # Enable OCSP stapling (server proactively fetches and caches certificate revocation status)
    ssl_stapling_verify on;                                                                   # Verify OCSP responses are signed by a trusted CA
    ssl_trusted_certificate /etc/nginx/ssl/chain.pem;                                         # CA certificate chain used to verify OCSP responses (must include intermediate + root)
    resolver 8.8.8.8 8.8.4.4 valid=300s;                                                      # DNS resolvers for OCSP lookups (Google Public DNS); cache results for 300 seconds
    resolver_timeout 5s;                                                                      # Timeout for DNS resolution (5 seconds max)

    # Add HSTS header
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;        # Instruct browsers to use HTTPS only for 1 year (31,536,000 seconds) on this domain and all subdomains
}

# Redirect HTTP to HTTPS
server {                                                                                       # Begin server block for HTTP redirects
    listen 80;                                                                                 # Listen on port 80 (unencrypted HTTP)
    server_name example.com www.example.com;                                                   # Match both apex and www subdomain (should align with your HTTPS server_name)
    return 301 https://example.com$request_uri;                                                # Permanent redirect (301) to HTTPS, preserving the original URI path
}
```

> **Note:** The original HTTP redirect block used only `www.example.com` as `server_name` while the HTTPS block used `example.com`. Both variants should be handled to avoid redirect loops or missed traffic.

**Security Headers**
```nginx
server {                                                                                                  # Begin server block for security headers
    add_header X-Frame-Options "SAMEORIGIN" always;                                                        # Prevent clickjacking by only allowing this site to be framed by itself
    add_header X-XSS-Protection "1; mode=block" always;                                                    # Legacy XSS filter for older browsers; note: deprecated and ignored by modern browsers (Chrome, Firefox)   rely on CSP instead
    add_header X-Content-Type-Options "nosniff" always;                                                     # Prevent Multipurpose Internet Mail Extensions(MIME) type sniffing; forces browser to use the declared Content Type
    add_header Referrer-Policy "no-referrer-when-downgrade" always;                                         # Send Referer header only when protocol security level stays the same (e.g., HTTPS  > HTTPS); omit on HTTPS  > HTTP
    add_header Content-Security-Policy "default-src 'self' https: data: blob: 'unsafe-inline'" always;     # CSP: restrict resources to same origin and HTTPS; note: 'unsafe-inline' weakens XSS protection,  consider using nonces or hashes for inline scripts in production
}
```

- Don't have a domain yet?
  
  - If you're testing or learning, generate a self signed certificate instead.
    
  - Create the SSL directory
```bash
sudo mkdir -p /etc/nginx/ssl/sslkey
sudo mkdir -p /etc/nginx/ssl/sslcert
```

  - Generate certificate
```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/sslkey/key.pem \
  -out /etc/nginx/ssl/sslcert/cert.pem
```

Parameters:
  * `-x509`: Output a self signed certificate instead of a certificate signing request (CSR)
  * `-nodes`: Do not encrypt the private key with a passphrase (stands for "no DES"; allows Nginx to load the key without a password prompt)
  * `-days 365`: Certificate validity period (365 days)
  * `-newkey rsa:2048`: Generate a new 2048-bit RSA key pair. RSA 2048 is still considered acceptable, though 3072 or 4096 bits is recommended for higher security or longer lived certificates
  * `-keyout`: Output path for the private key file
  * `-out`: Output path for the certificate file
  * `-subj`: Set certificate subject fields directly (bypasses interactive prompts)
    * Example: `-subj "/C=US/ST=California/L=San Francisco/O=MyOrg/CN=localhost"`

- Generate Self-Signed Certificate
  - Create the certificate directory:
```bash
sudo mkdir -p /etc/pki/tls/private
sudo mkdir -p /etc/pki/tls/certs
```

  - Generate the self signed certificate:
```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/pki/tls/private/localhost.key \
    -out /etc/pki/tls/certs/localhost.crt \
    -subj "/C=US/ST=State/L=City/O=Organization/CN=localhost"
```

- Configure Nginx for SSL
  
  - Create the Nginx configuration file:
    
```bash
sudo vim /etc/nginx/conf.d/ssl-website.conf
```

```nginx
server {
    listen 443 ssl;
    server_name 127.0.0.1;

    root /usr/share/nginx/html;

    ssl_certificate     /etc/pki/tls/certs/localhost.crt;
    ssl_certificate_key /etc/pki/tls/private/localhost.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;
    add_header X-Content-Type-Options nosniff always;
    add_header X-Frame-Options DENY always;

    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    access_log  /var/log/nginx/ssl_access.log;
    error_log   /var/log/nginx/ssl_error.log;
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name 127.0.0.1;
    return 301 https://$host$request_uri;
}
```

>**Note:** `max-age=63072000` in the HSTS header equals 2 years (63,072,000 seconds). The `preload` directive nominates the domain for browser HSTS preload lists   only use it with a real domain, not `127.0.0.1` or `localhost`, as preload does not apply to IP addresses or loopback addresses.

    document.getElementById('protoSpan').textContent = loc.protocol;
    document.getElementById('hostSpan').textContent  = `//${loc.hostname}`;
  }

  function updateDetails() {
    document.getElementById('protocol').textContent     = loc.protocol;
    document.getElementById('hostname').textContent     = loc.hostname;
    document.getElementById('port').textContent         = getPort();
    document.getElementById('secureContext').textContent = window.isSecureContext ? 'Yes' : 'No';
    document.getElementById('pageUrl').textContent      = loc.href;
  }

  function refresh() {
    updateStatus();
    updateDetails();
    const t = document.getElementById('toast');
    t.classList.add('show');
    setTimeout(() => t.classList.remove('show'), 2200);
  }

  function openModal() {
    document.getElementById('tlsSupport').textContent = window.isSecureContext ? 'Supported & Active' : 'Not Active';
    document.getElementById('userAgent').textContent  = navigator.userAgent;
    document.getElementById('timestamp').textContent  = new Date().toLocaleString();
    document.getElementById('modal').classList.add('open');
  }

  function closeModal()      { document.getElementById('modal').classList.remove('open'); }
  function handleBackdrop(e) { if (e.target === document.getElementById('modal')) closeModal(); }

  updateStatus();
  updateDetails();

**Observe**

- Try opening: `http://localhost`
  
  - **What happens?** The browser is automatically redirected to `https://localhost` with a `301 Moved Permanently` response. This occurs because the HTTP server block in the configuration matches port 80 requests and issues a `return 301` redirect to the HTTPS equivalent URI.
    
  - **Why?** The `return 301 https://$host$request_uri;` directive tells Nginx to permanently redirect any unencrypted HTTP request to the HTTPS version. After the first redirect, browsers that support HSTS will also refuse to connect over HTTP for the duration of the `max-age` period, issuing the redirect internally without even contacting the server.
    
>**Note:** Because this uses a self signed certificate, the browser will still show a certificate warning (`NET::ERR_CERT_AUTHORITY_INVALID` or similar). This is expected;   the certificate is not signed by a trusted CA. You can bypass this in a browser during testing, but it confirms that the redirect and TLS termination are working correctly.
