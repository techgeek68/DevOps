# Jenkins for CI/CD

---

Prerequities:
> Create a VM Named `Jenkins Master`

**Introduction**

Jenkins is an open source automation server widely used for Continuous Integration (CI) and Continuous Delivery/Deployment (CD). It automates the building, testing, and deployment of software projects, significantly accelerating development cycles.

**Features**
  - **Automated Builds & Testing**: Compiles code, runs tests, and reports issues automatically on code commits
  - **Extensible Plugins**: 1,900+ plugins for version control (Git, SVN), cloud platforms (AWS, Azure), containers (Docker), and more
  - **Pipeline as Code**: Define CI/CD workflows using `Jenkinsfile` (declarative/scripted syntax)
  - **Distributed Architecture**: Scale workloads across multiple worker nodes
  - **Monitoring & Notifications**: Track build status via dashboards and alerts (email, Slack)

**Uses**
  - **Continuous Integration**: Automatically test code changes on every commit
  - **Continuous Deployment**: Deploy validated code to production environments
  - **Scheduled Tasks**: Run nightly builds, cleanup jobs, or periodic maintenance
  - **Multi Environment Testing**: Test across different OS, browsers, or dependency versions

**Alternatives to Jenkins**
  * Hudson, TeamCity, Bamboo, GitHub Actions, GitLab CI, CircleCI, Azure Pipelines

---

**What is Jenkins?**

Jenkins orchestrates the entire software delivery lifecycle through **jobs**, **nodes**, and **pipelines**.

**Basic Terminology**

| Term         | Description                                                        |
|--------------|--------------------------------------------------------------------|
| **Job**      | A task Jenkins performs (e.g., build, test, deploy)                |
| **Node**     | A machine (master/agent) that executes tasks                      |
| **Executor** | A slot on a node for running a job                                |
| **Artifact** | Output files from a build (e.g., JARs, WARs, logs)               |
| **Trigger**  | An event that starts a job (e.g., SCM commit, cron schedule)      |

**CI/CD Concepts**

CI/CD is a set of practices that automate software development workflows to deliver code changes faster, more reliably, and with higher quality.


**CI: Continuous Integration**

  * A practice where developers frequently (often multiple times a day) merge their code changes into a shared repository.

**Primary Goal:** Catch bugs early and prevent integration conflicts, ensuring the codebase remains stable and reliable.

**Features:**

- **Auto Builds**: The system automatically compiles the code after each merge
- **Auto Testing**: Unit and integration tests run automatically on every change to validate code correctness


**CD: Continuous Delivery**

  * A practice where tested code is automatically deployed to a staging or pre-production environment.
  * Primary Goal is to ensure that the application is always in a deployable state and ready for production release.

**Features:**
  - **Automation to Staging**: After passing automated tests, code is deployed to a non-production environment
  - **Manual Approval**: Required before deploying to production


**CD: Continuous Deployment**
  * An advanced CI/CD practice where code changes are automatically released to production after passing all tests.
  * Primary Goal is to deliver features and fixes to users within minutes or hours of development.

```
Developer → Code Commit → Build → Test → Stage → (Auto/Manual) → Production
   CI ──────────────────────────────┘         │
   Continuous Delivery ───────────────────────┘ (manual gate)
   Continuous Deployment ─────────────────────── (fully automated)
```

**Jenkins Architecture: Master & Slave**

Jenkins uses a distributed architecture consisting of a Controller (master) and one or more Agents (slaves).

  - **Controller**: Central server that manages pipelines, schedules jobs, and stores configurations
  - **Agent**: Worker nodes that execute jobs, distributing the load across machines

```

                              +-------------------+
                              |     Controller    |
                              |      (Jenkins)    |
                              +---------+---------+
                                        |
        ---------------------------------------------------------------
        |                    |                   |                     |
       SSH                  SSH                 SSH                   SSH
        |                    |                   |                     |
+---------------+   +---------------+   +---------------+     +---------------+
|    Agent 1    |   |    Agent 2    |   |    Agent 3    |     |    Agent 4    |
|   OS: Linux   |   |   OS: Linux   |   |   OS: Linux   |     |  OS: Windows  |
| Tools: JS     |   | Tools: Python |   | Tools: Python |     | Tools: .NET   |
+---------------+   +---------------+   +---------------+     +---------------+
```

## Installation & Configuration

**Installing Jenkins**

**Step 1: Install Java (JDK 11+)**

```bash
sudo dnf install java-21-openjdk-devel
```

**Step 2: Set Java Path**

```bash
vim ~/.bashrc
```

Add the following lines:

```bash
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH=$PATH:$JAVA_HOME/bin
```

Apply the changes:

```bash
source ~/.bashrc
echo $JAVA_HOME
```

- List the repository packages to find out whether Jenkins is available or not:

```bash
dnf list jenkins
```

**Building CI/CD Pipelines**

**Java Apps: Deployment to Tomcat**

This pipeline pulls Java source code from Git, builds it with Maven, and deploys the resulting WAR/JAR to an Apache Tomcat server.

**Pipeline Flow**

```
GitHub Repo → Jenkins (Git Pull) → Maven Build → WAR/JAR Artifact → Deploy to Tomcat
```

**Prerequisites**
  - Git installed on the Jenkins server
  - Maven installed and configured in Jenkins
  - Tomcat server running and accessible
  - **Plugins:** Git, Maven Integration, Deploy to Container


**Example Declarative Pipeline (`Jenkinsfile`)**

```groovy
pipeline {
    agent any

    tools {
        jdk 'java'
        maven 'maven'
    }

    stages {
        stage('Clone Repository') {
            steps {
                git branch: 'main', url: 'https://github.com/your-org/your-java-app.git'
            }
        }

        stage('Build with Maven') {
            steps {
                sh 'mvn clean package'
            }
        }

        stage('Run Tests') {
            steps {
                sh 'mvn test'
            }
        }

        stage('Deploy to Tomcat') {
            steps {
                deploy adapters: [
                    tomcat9(
                        url: 'http://your-tomcat-server:8080',
                        credentialsId: 'tomcat-credentials'
                    )
                ],
                war: 'target/*.war',
                contextPath: '/your-app'
            }
        }
    }

    post {
        success {
            echo 'Build and deployment successful!'
        }
        failure {
            echo 'Build or deployment failed.'
        }
    }
}
```

**PHP/Python Apps: Deployment to Apache/Nginx**

This pipeline pulls PHP or Python application code from Git and deploys it to an Apache or Nginx web server using SSH.

**Pipeline Flow**

```
GitHub Repo → Jenkins (Git Pull) → Test → Deploy via SSH → Apache/Nginx Server
```

**Prerequisites**
  - Git installed on the Jenkins server
  - Web server (Apache/Nginx) running on the target machine
  - SSH access configured between Jenkins and the target server
  - **Plugin:** Publish Over SSH

**Example: PHP App Deployment to Apache**

```groovy
pipeline {
    agent any

    stages {
        stage('Clone Repository') {
            steps {
                git branch: 'main', url: 'https://github.com/your-org/your-php-app.git'
            }
        }

        stage('Deploy to Apache via SSH') {
            steps {
                sshPublisher(publishers: [
                    sshPublisherDesc(
                        configName: 'apache-server',
                        transfers: [
                            sshTransfer(
                                sourceFiles: '**/*',
                                remoteDirectory: '/var/www/html/your-app',
                                removePrefix: ''
                            )
                        ]
                    )
                ])
            }
        }
    }

    post {
        success {
            echo 'PHP app deployed to Apache successfully!'
        }
        failure {
            echo 'Deployment failed.'
        }
    }
}
```

**Example: Python App Deployment to Nginx**

```groovy
pipeline {
    agent any

    stages {
        stage('Clone Repository') {
            steps {
                git branch: 'main', url: 'https://github.com/your-org/your-python-app.git'
            }
        }

        stage('Install Dependencies') {
            steps {
                sh '''
                    python3 -m venv venv
                    . venv/bin/activate
                    pip install -r requirements.txt
                '''
            }
        }

        stage('Run Tests') {
            steps {
                sh '''
                    . venv/bin/activate
                    pytest tests/
                '''
            }
        }

        stage('Deploy to Nginx via SSH') {
            steps {
                sshPublisher(publishers: [
                    sshPublisherDesc(
                        configName: 'nginx-server',
                        transfers: [
                            sshTransfer(
                                sourceFiles: '**/*',
                                remoteDirectory: '/var/www/your-python-app',
                                execCommand: '''
                                    cd /var/www/your-python-app
                                    source venv/bin/activate
                                    sudo systemctl restart your-app.service
                                '''
                            )
                        ]
                    )
                ])
            }
        }
    }

    post {
        success {
            echo 'Python app deployed to Nginx successfully!'
        }
        failure {
            echo 'Deployment failed.'
        }
    }
}
```

**User & Role Management**

Jenkins supports fine-grained access control to manage who can view, configure, and execute pipelines.

**Setting Up User Authentication**
  1. Navigate to `Manage Jenkins` → `Security`
  2. Under **Security Realm**, select:
   - **Jenkins' own user database** — for local user management
   - **LDAP** — for enterprise directory integration


**Role Based Access Control (RBAC)**

  **Plugin Required:** `Role Based Authorization Strategy`

  **Method:** `Manage Jenkins` → `Security` → `Authorization` → `Role-Based Strategy`

**Steps to Configure**
  1. **Install** the Role Based Authorization Strategy plugin
  2. **Enable** it under `Manage Jenkins` → `Security` → `Authorization`
  3. **Define Roles** under `Manage Jenkins` → `Manage and Assign Roles`:

| Role        | Permissions                                  |
|-------------|----------------------------------------------|
| Admin       | Full access to all Jenkins settings and jobs |
| Developer   | Create, configure, and run jobs              |
| Viewer      | Read-only access to job statuses and logs    |
| Deployer    | Run deployment pipelines only                |

4. **Assign Users** to their respective roles


**Permission Categories**
  - **Overall**: Administer, Read, RunScripts
  - **Job**: Build, Cancel, Configure, Create, Delete, Read, Workspace
  - **View**: Configure, Create, Delete, Read
  - **Credentials**: Create, Delete, Manage Domains, Update, View



**Key File Paths**

