# Load Balancer demonstration (Using `podman` on RHEL/CentOS/Fedora)

## Architecture

```
Your Fedora PC
├── Container: lb        (Load Balancer   Nginx) > Port 8080
├── Container: node1     (Backend Server 1)
└── Container: node2     (Backend Server 2)
```

> Note: Fedora comes with Podman pre-installed. It works exactly like Docker but is daemonless and rootless by default, perfect for local lab setups without VMs.

---

## Step 1: Install Podman

- Check if Podman is already installed
```bash
podman --version
```
- If not installed, install it
```bash
sudo dnf install podman -y
```

- Verify
```bash

```

## Step 2: Create a Custom Network

```bash
podman network create lb-network
```

> This creates an isolated network where containers can communicate with each other using their container names as hostnames (e.g., `node1`, `node2`)   no need to hardcode IP addresses.

## Step 4: Configure Node 1 (Backend Server 1)

* Create the HTML page for Node 1:

```bash
cat > ~/lb-demo/node1/index.html << 'EOF'
<!DOCTYPE html>
<html>
<head>
  <title>Backend Server</title>
</head>
<body>
  <h1>Backend Server</h1>
  <h2>This is a load balancer test page</h2>
  <p>Served by node1.</p>
</body>
</html>
EOF
```

* Create the Nginx config for Node 1:

```bash
cat > ~/lb-demo/node1/backend.conf << 'EOF'
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html/devops;
    index index.html;

    access_log /var/log/nginx/backend_access.log;
    error_log /var/log/nginx/backend_error.log;

    location / {
        try_files $uri $uri/ =404;
    }

    location /health {
        access_log off;
        default_type text/plain;
        return 200 "healthy\n";
    }
}
EOF
```

## Step 6: Configure Load Balancer

* Create the Nginx config for the Load Balancer:

```bash
cat > ~/lb-demo/lb/loadbalancer.conf << 'EOF'
upstream backend_servers {
    # Basic round-robin (default)
    server node1:80;
    server node2:80;
}

server {
    listen 80;
    server_name _;

    access_log /var/log/nginx/lb_access.log;
    error_log /var/log/nginx/lb_error.log;

    location / {
        proxy_pass http://backend_servers;

        # Essential proxy headers
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeout settings
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }
}
EOF
```

> **Note:** Container names (`node1`, `node2`) are used instead of IP addresses. Podman's internal DNS resolves them automatically within the `lb-network`.

## Step 8: Start Backend Containers

* Start Node 1:

```bash
podman run -d \
  --name node1 \
  --network lb-network \
  -v ~/lb-demo/node1/index.html:/usr/share/nginx/html/devops/index.html:ro,z \
  -v ~/lb-demo/node1/backend.conf:/etc/nginx/conf.d/default.conf:ro,z \
  nginx:alpine
```

| Command | Meaning |
|---|---|
| `podman run` | Create and start a container |
| `-d` | Run in the background |
| `--name node1` | Name the container `node1` |
| `--network lb-network` | Connect to `lb-network` |
| `-v` | Mount a file from host into the container |
| `host path:container-path` | Left: your machine, Right: inside container |
| `ro` | Read only (container cannot modify the file) |
| `z` | SELinux: allow the container to read the file |
| `nginx:alpine` | Image to use |

## Step 9: Start Load Balancer Container

```bash
podman run -d \
  --name lb \
  --network lb-network \
  -p 8080:80 \
  -v ~/lb-demo/lb/loadbalancer.conf:/etc/nginx/conf.d/default.conf:ro,z \
  nginx:alpine
```

| Part | Meaning |
|---|---|
| `podman run` | Create and start a container |
| `-d` | Run in the background |
| `--name lb` | Name the container `lb` |
| `--network lb-network` | Connect to `lb-network` |
| `-p 8080:80` | Map port 8080 on your machine to port 80 inside the container |
| `-v` | Mount a file from host into the container |
| `~/lb-demo/lb/loadbalancer.conf` | Config file on your machine |
| `/etc/nginx/conf.d/default.conf` | Where it appears inside the container |
| `ro` | Read only |
| `z` | SELinux: allow the container to read the file |
| `nginx:alpine` | Image to use |

**`-p`**
```
8080 : 80
 │      └─ Port inside the container
 └─ Port on your machine
```

## Comparison: VM Setup vs Podman Setup

| Feature | VM Setup | Podman/Container Setup |
|---|---|---|
| Servers | 3 separate VMs | 3 containers on 1 machine |
| IP Addressing | Static IPs (e.g., 10.10.0.6/7/8) | Container names (node1, node2) |
| Firewall config | Required on each VM | Only `-p 8080:80` port mapping |
| Resource usage | High (RAM/CPU per VM) | Lightweight |
| Setup time | 30+ minutes | ~5 minutes |
| Concept learned | Same (Nginx LB + Round Robin) | Same (Nginx LB + Round Robin) |
